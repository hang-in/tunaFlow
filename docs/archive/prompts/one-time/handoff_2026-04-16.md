## 2. 절대 지켜야 할 규칙 (Hard Rules)

### 2.1 Git stash 금지
오늘 두 번 사고 발생. `git stash push -u` 로 untracked 파일(문서/아이디어)이 묻혔다가 `git checkout` 후 워킹트리에서 사라진 것처럼 보임 → 사용자 혼란. **stash 쓰지 말 것. 필요하면 명시적으로 새 브랜치 만들어서 커밋하고 이동.**

### 2.2 DB 건드리는 위험한 명령 금지
- 사용자 환경의 **AppCleaner 가 bundle id 경로 폴더를 통째로 휴지통으로 보내는 패턴** 존재 → DB 날아가는 사고 두 번. 이제 dev DB 는 `~/.tunaflow/db/tunaflow.db` (AppCleaner 영향권 밖) 이지만, 혹시 경로 이동/삭제 제안 시 **사용자 동의 없이 실행 금지**.
- 영구 백업 위치: `~/tunaflow-db-backups/tunaflow-2026-04-16T0721-recovery.db` (37M, projects 7/conv 233/msg 2557). 유사시 여기서 복구.

### 2.3 쿼터 절약 원칙
- **Opus 호출 최소화**. 코드 읽을 때 grep/Read 를 쪼개서 필요한 부분만.
- 큰 PR 한 방 대신 **작은 PR 순차 머지**. 실패 시 롤백 싸고 재실행 적음.
- 매 PR 후 `git checkout main && git pull --ff-only`.
- **Haiku 사용 가능한 곳은 Haiku** (memory_compression 이미 그렇게 됨).
- 확인만 필요한 요청에는 코드 수정 절대 금지. 사용자가 명시적으로 "수정"이라고 하면 그때.

### 2.4 빌드/dev 전환 시
- dev 는 `~/.tunaflow/db/tunaflow.db`, release 는 `com.tunaflow.app/tunaflow.db` 를 봄 (완전 분리됨)
- release 빌드 테스트 시 `./scripts/build.sh --wipe-sandbox --remove-previous-app` 로 깨끗한 온보딩 재현
- dev 실행 중에는 AppCleaner 가 `com.tunaflow.app/` 를 지우더라도 dev DB 영향 없음

### 2.5 사용자 협업 스타일
- **한국어 존댓말**
- 마케팅 톤/과장 표현 싫어함. 솔직하고 직접적
- 결정 빠름. 질문에 옵션 2~3개 주면 바로 선택
- 실측 기반. 가설만 말하면 "확인해봐" 요구
- 이미 결정된 것 재논의 싫어함 (플랜 문서에 쓰인 방향은 그대로 진행)
- "수정 금지, 확인만"이라 말하면 **절대 코드 변경 X**

## 4. 알려진 이슈 (Known Issues)

### 4.1 윈도우 스냅 + 창 위치 저장 (한 뿌리 가능성)
**증상**:
- 앱을 macOS 화면 모서리에 갖다 대도 edge tiling 안 됨
- Magnet/Rectangle 같은 3rd-party 스냅 도구도 tunaFlow 창 무시
- 앱 종료 시 창 위치/크기 저장 안 됨 (다음 실행 시 기본 위치 복귀)

**가설**: `titleBarStyle: "Overlay" + hiddenTitle: true` 조합이 NSWindow 의 `NSWindowStyleMaskTitled` 비트를 꺼뜨려, macOS 의 표준 window-autosave + edge-tiling 둘 다 불가능.

**검증 방법** (권장):
1. `tauri.conf.json` 에서 `titleBarStyle` 필드 제거 (default) → 표준 타이틀바 복귀
2. 스냅 + 창 위치 저장이 둘 다 복원되는지 확인
3. 확인되면 원인 = titleBarStyle. 해결은 Rust side 에서 `NSWindow` 속성 조작 (`window-ext` crate 또는 raw objc). 커스텀 TitleBar UI 유지하면서 OS 표준 동작 회복.

**중요**: 사용자가 **"한동안 스냅 정상 동작했었다"** 언급 → 특정 커밋/Tauri 업그레이드 후 regression. git log 에서 `tauri.conf.json`, `tauri-plugin-window-state`, TitleBar 컴포넌트 관련 커밋 찾으면 정확한 회귀 지점 포착 가능.

**우선순위**: P1 (베타 차단 아님, 베타 후).

### 4.2 Reviewer lean ContextPack (S2.5)
review branch 의 첫 RT 프롬프트에 불필요한 retrieval / cross-session / compressed-memory 가 5~10k 토큰 낭비. `context_mode_override = "standard"` 로 해결 가능하지만 Conventions Sync 가 더 근본 해결이라 함께 처리 예정.

### 4.3 Marker UX 플래시
스트리밍 중 `<!-- tunaflow:... -->` 가 텍스트로 잠깐 노출되던 문제. `MessageItem.tsx` 에 `hideTrailingIncompleteComment` 추가로 부분 해결. 단 `PlanProposalCard` / `RtMessageCard` 등 다른 렌더 경로에는 미적용 → 그곳에서 여전히 플래시 가능. 필요 시 동일 헬퍼 적용.

### 4.4 Onboarding 스캔 — 일부 스택 (Zig/Haskell 등)
manifest 후보 32종으로 커버리지 넓혔지만 여전히 놓칠 가능성 있음 (`agda.yaml` 등). 증상 보고되면 추가.

### 4.5 LMStudio 감지
timeout 3s 로 올렸고 eprintln 디버그 로그 있음. 여전히 감지 실패 시 앱 콘솔에서 `[agent-detect] lmstudio` 로 시작하는 로그 확인.

## 9. 여담

사용자와 **Gemini 와의 22분 배틀**은 오늘 세션의 하이라이트였다. 다음 세션이 비슷한 외부 LLM 리뷰 국면을 만나면 `docs/reference/geminiCriticReview_2026-04-15.md` 의 **검증 패턴**을 그대로 쓰면 된다:

> 1. 외부 LLM 에게 "신랄하게 + 시니어" 같은 단어로 요청하지 말 것 (단정 안전벨트 풀림)

> 3. 반박은 한 라운드만 — 2라운드 가면 회피 모드로 들어감
> 4. 진짜 타당한 지적 1~2건만 건져서 플랜에 반영
