# tunaFlow 문서 IA / 거버넌스 계획

- 작성자: OpenAI Codex
- 작성 시각: 2026-03-30
- 상태: 초안

## 목적

문서가 많아질수록 에이전트가 모든 문서를 읽는 방식은 실패한다.
이 계획의 목적은 `docs/`를 줄이는 것이 아니라,
에이전트와 사람이 필요한 문서만 빠르게 찾고 신뢰할 수 있게 만드는 규칙을 정하는 것이다.

## 핵심 판단

문서 문제는 양보다 탐색성과 신뢰성 문제다.

따라서 우선순위는:

1. 문서 역할 분리
2. index 개선
3. 상태 라벨 정리
4. 아카이브 분리

## 현재 문제

### 1. 같은 기능에 문서가 여러 개 있다

- plan
- prompt
- reference
- how-to

가 섞이기 쉽다.

### 2. 현재 문서와 과거 문서가 함께 보인다

아카이브 문서나 초안 문서가 현재 기준처럼 읽힐 수 있다.

### 3. 읽는 순서가 고정돼 있지 않다

새 세션 에이전트가 어떤 문서를 먼저 읽어야 할지 매번 추론해야 한다.

## 목표

### 1. 문서 타입별 역할 고정

- `reference` = 현재 기준 사실
- `plans` = 앞으로 할 일
- `prompts` = 실행 지시문
- `how-to` = 운영 절차
- `archive` = 과거 문서

### 2. 인덱스 강화

각 폴더 `index.md`는 단순 목록이 아니라:

- 무엇이 핵심인지
- 무엇이 최신인지
- 무엇이 아카이브인지
- 무엇부터 읽어야 하는지

를 알려야 한다.

### 3. 문서 상태 가시화

문서 상단 상태:

- 초안
- 진행 중
- 부분 완료
- 완료
- 아카이브

### 4. 작업별 추천 읽기 묶음

예:

- 새 세션 시작
- UI 작업
- Rust command 작업
- engine parity 작업
- settings/agent 작업

각 묶음은 3~5개 문서로 제한하는 것이 좋다.

## 권장 실행 순서

### Phase 1. 문서 IA 기준 고정

- 내비게이션 모델 문서 작성
- 문서 타입별 역할 확정
- index에 읽기 순서 추가

### Phase 2. 상태 라벨 보강

- 핵심 문서 상단에 상태 통일
- 아카이브 문서 명시 강화

### Phase 3. 작업별 reading pack 정리

- 예: `Agent/Profile pack`, `Chat UI pack`, `Engine parity pack`

### Phase 4. 낡은 문서 정리

- archive 이동
- 경고 표기

## 비목표

- 모든 문서 이름 변경
- 대규모 폴더 재배치
- 문서 내용 전면 재작성

## 완료 기준

1. 에이전트가 새 작업을 시작할 때 어떤 문서부터 읽어야 할지 명확하다
2. 현재 문서와 아카이브 문서가 쉽게 구분된다
3. index만 봐도 문서 탐색이 가능하다
