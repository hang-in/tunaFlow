# Prompt Quality Adoption Plan

상태: 제안
작성: 2026-03-30

## 목표

`Prompt Authoring Policy`를 이후 새 Claude 프롬프트 작성의 기본 규칙으로 적용한다.

## 적용 범위

- 새로 만드는 `docs/prompts/*.md`
- 자주 재사용되는 템플릿 프롬프트
- 나중에 tunaFlow 내부 prompt preset/template 기능

## 1차 적용 원칙

### 새 프롬프트부터 적용

- 기존 프롬프트 전면 소급 수정은 하지 않는다
- 이후 새 프롬프트는 policy 기준으로 작성한다

### 핵심 작업 프롬프트 우선 보강

아래처럼 품질 영향이 큰 문서부터 필요 시 보강한다.

- ContextPack 관련 프롬프트
- context-hub / flow agent 관련 프롬프트
- 대규모 리팩토링 프롬프트

## 최소 체크리스트

새 프롬프트를 만들 때 아래를 확인한다.

- 프로젝트 경로가 있는가
- 먼저 읽을 문서가 있는가
- 먼저 확인할 파일이 구체적인가
- 작업 시작 전 의견 항목이 있는가
- 구현 범위와 비목표가 분리되어 있는가
- 검증 항목이 있는가
- 출력 형식이 있는가
- 필요 시 한국어 고정 문구가 있는가
- 필요 시 skill 지시가 있는가

## 메모

이 계획은 프롬프트를 “많이 만드는 것”보다 “일관되게 강하게 만드는 것”을 우선한다.
